# 折弯预览功能整理规划

## 1. 目标

本文档用于整理 `折弯预览` 相关功能的现状、已经确认的问题、根因、当前实际实现链路，以及后续收口顺序。

当前目标不是一次性重写全部逻辑，而是先把问题分层，明确：

1. 当前代码链路里，哪些功能已经具备。
2. 目前真正影响稳定性的结构问题是什么。
3. 当前实现是怎样串起来的。
4. 实际调试中遇到过哪些问题，已经如何处理。
5. 后续应该优先改哪里，避免继续在局部补丁上反复返工。

适用范围：

- `HMI/JSZW1000A/JSZW1000A/SubWindows/SubOPAuto1.cs`
- `HMI/JSZW1000A/JSZW1000A/SubWindows/SubOPAutoSet.cs`
- `HMI/JSZW1000A/JSZW1000A/SubWindows/SubOPManual.cs`
- `HMI/JSZW1000A/JSZW1000A/SubWindows/SubOPAutoView.cs`
- `HMI/JSZW1000A/JSZW1000A/MainFrm.cs`
- `HMI/JSZW1000A/JSZW1000A/MainFrm.SemiAuto.cs`
- `HMI/JSZW1000A/JSZW1000A/MainFrm.SemiAuto.DerivedState.cs`
- `HMI/JSZW1000A/JSZW1000A/MainFrm.ManualSemiAuto.cs`

## 2. 当前职责划分

### 2.1 `SubOPAuto1`

职责：

- 编辑工作单图形。
- 修改 `lengAngle`、挤压、边做边分切、颜色面、顺序等基础参数。
- 根据当前图形重建 `pxList`。

说明：

- 图形修改后，会通过 `MarkSemiAutoSequenceStale()` 把旧半自动步骤表标记为失效。
- `CalculatePhysicalPoints()` 当前会参考旧 `CurtOrder.pxList` 保持显示朝向连续，这个策略本身不是 bug，但属于后续可显式化的状态。

### 2.2 `MainFrm`

职责：

- 维护 `CurtOrder`。
- 负责页面切换。
- 负责 `create生产序列()`。
- 负责将当前步骤表发送到半自动。

说明：

- `MainFrm` 是 `set/view/manual` 三页共享数据的统一入口。

### 2.3 `SubOPAutoSet`

职责：

- 作为唯一的折弯设置页面。
- 显示板型布置图。
- 按当前 `lstSemiAuto` 生成折弯按钮和参数面板。
- 对步骤顺序和工艺参数进行编辑。

当前已具备：

- 折弯按钮生成。
- 当前步骤选中与参数回显。
- 前移/后移。
- 抓取类型、松开高度、内外选择、折弯方向、回弹值编辑。
- 修改后统一回写 `CurtOrder.lstSemiAuto`。
- 修改后统一调用 `RebuildSemiAutoDerivedState(ref CurtOrder)`。

### 2.4 `SubOPManual`

职责：

- 表格方式显示和编辑半自动步骤表。
- 支持插入、删除、复制、调整工艺参数。

说明：

- 表格编辑最终也回写 `CurtOrder.lstSemiAuto`。
- 当前与 `set` 一样，最终依赖统一派生状态重建链路。

### 2.5 `SubOPAutoView`

职责：

- 作为唯一的折弯预览页面。
- 显示点动预览和自动预览。
- 根据当前步骤表绘制阶段板型、颜色线、挤压图元和翻面完成态。

说明：

- 旧版混在 `view` 中的“折弯布置/设置”逻辑已经基本收掉。
- 当前 `view` 已经不再承担 `set` 的编辑职责。

## 3. 当前已落地的功能

1. `MainFrm.切入折弯预览(true/false)` 已经拆分出两条路径：
   - `true` 进入 `SubOPAutoView`
   - `false` 进入 `SubOPAutoSet`
2. `SubOPAutoSet` 已经可以完整编辑当前折弯步骤表。
3. `view` 进入时，优先使用当前 `CurtOrder.lstSemiAuto`，不再默认覆盖人工编辑结果。
4. `RebuildSemiAutoDerivedState(ref CurtOrder)` 已经成为后挡、`is色下`、操作提示等派生状态的统一重建入口。
5. `SubOPAutoView` 已经收口到统一预览刷新链路：
   - `BuildPreviewSegmentsForDrawStep()`
   - `redrawPreView()`
   - `UpdatePreviewStepInfo()`
6. 预览页已经改成 `Bitmap -> Paint/Invalidate` 模式，不再在主链路里直接使用 `CreateGraphics()`。

## 4. 当前实际实现链路

### 4.1 构图数据源

- 上游基础数据是 `CurtOrder.lengAngle` 与 `CurtOrder.pxList`。
- `SubOPAuto1` 负责修改这两份数据。
- 一旦图形变更，旧半自动步骤表会被标记失效。

### 4.2 半自动步骤生成链路

- `MainFrm.create生产序列()` 负责从 `lengAngle + pxList` 生成 `CurtOrder.lstSemiAuto`。
- 生成顺序是：
  1. 先处理两端挤压/开口挤压。
  2. 再生成中间普通折弯。
  3. 最终写入每步的 `折弯序号 / 行动类型 / 折弯方向 / 长角序号 / 坐标序号 / 内外选择 / 后挡位置`。

### 4.3 Set 编辑链路

- `SubOPAutoSet` 进入后按当前 `lstSemiAuto` 生成布置按钮。
- 用户在 `set` 页做的调序和参数编辑直接修改 `CurtOrder.lstSemiAuto`。
- 每次修改后统一调用：
  - `RebuildSemiAutoDerivedState(ref CurtOrder)`
  - `MarkSemiAutoStepsManuallyEdited()`

### 4.4 派生状态重建链路

- `MainFrm.SemiAuto.DerivedState.cs` 负责：
  - 重建 `后挡位置`
  - 重建 `is色下`
  - 重建 `操作提示`
  - 提供阶段板型快照
- 当前存在两种阶段板型接口：
  - `BuildSemiAutoStageProfile()`：给后挡和派生状态使用。
  - `BuildSemiAutoPreviewStageProfile()`：给 `view` 预览使用。

### 4.5 View 预览链路

- `MainFrm.切入折弯预览(true)` 是 `view` 入口。
- `SubOPAutoView.RefreshPreviewState()` 会先做数据源校验：
  1. 如果当前步骤表是人工编辑结果，则直接保留并重建派生状态。
  2. 如果当前步骤表几何元数据无效，则先重建生产序列。
  3. 然后进入预览页面。
- `SubOPAutoView` 当前的点动预览流程：
  1. `PreViewSt()` 推进 `DrawStep`
  2. `BuildPreviewSegmentsForDrawStep()` 按当前步骤表重建当前阶段板型
  3. `redrawPreView()` 画主折线、颜色线、挤压图元
  4. `UpdatePreviewStepInfo()` 刷新当前步和操作提示

### 4.6 翻面预览链路

- 翻面已改成“翻面完成态”模式。
- 当某一步之后需要翻面时：
  1. 先进入一帧“翻面完成态”
  2. 图形按当前步骤表重建
  3. 之后继续正常推进下一步

## 5. 实际调试中遇到的问题与处理方式

### 5.1 Set 调序后，View 仍按旧步骤预览

现象：

- 在 `set` 页调整步骤顺序后，`view` 进入时又恢复成旧顺序或旧图形。

根因：

- 预览入口会在某些情况下直接调用 `create生产序列()`，覆盖掉 `set` 当前编辑结果。

处理方式：

- 引入 `HasManualSemiAutoEdits()`。
- 进入预览页时，优先保留并使用当前 `CurtOrder.lstSemiAuto`。

### 5.2 后挡计算在调序后漂移

现象：

- 将某一步移动到后面后，后挡值变成 `0` 或明显左右错位。

根因：

- 旧算法把旧 `后挡位置` 当成了新几何的输入，形成循环依赖。

处理方式：

- 引入统一派生状态重建函数 `RebuildSemiAutoDerivedState(ref CurtOrder)`。
- 后挡位置、`is色下`、操作提示统一改为派生状态。

### 5.3 View 和 Set 职责混杂

现象：

- `SubOPAutoView` 同时保留了“布置/设置”和“预览”两套逻辑。
- 页面切换后容易出现错位、不显示、状态不同步。

处理方式：

- `SubOPAutoSet` 收口为唯一设置页。
- `SubOPAutoView` 收口为唯一预览页。
- 旧布置逻辑逐步从 `view` 中移除。

### 5.4 `CreateGraphics()` 导致预览不稳定

现象：

- 页面切换、遮挡恢复、重复进入后，图形容易丢失或残留。

处理方式：

- 主预览链路改成 `Bitmap -> Paint/Invalidate`。
- 隐藏测试按钮 `button2` 一并清理。

### 5.5 翻面预览出现“未折部分在动”或“左右反了”

现象：

- 翻面后看到未折部分在动、已折部分不动。
- 后续步骤整体左右反转。

处理方式：

- 引入“翻面完成态”。
- 翻面不再基于旧 `pxDraw` 增量旋转。
- 整板镜像限制在翻面完成态这一帧使用，不再持续作用到所有后续步骤。

### 5.6 双端挤压长度不一致

现象：

- 两端都配置 `15mm` 挤压，但第二次挤压在预览中明显比第一次长。

根因：

- 挤压步骤被并入阶段板型推进，第一次挤压长度被累计到后续步骤。

处理方式：

- 预览专用阶段板型中，挤压占位折弯与挤压步骤都不再推进折弯角状态。
- 挤压单独作为图元绘制，直接读取 `lengAngle[0] / lengAngle[99]` 的真实长度。

### 5.7 旧工单 `SemiAutoList` 元数据无效

现象：

- 某些旧 `ini` 工单导入后，正常折弯步的 `长角序号/坐标序号` 全是 `0`。
- 结果预览无法知道每一步真正折在哪里。

处理方式：

- 增加 `HasValidSemiAutoGeometryData()`。
- 进入预览时，如果步骤表几何元数据无效，则自动重建步骤表。

### 5.8 调试过程中出现乱码字段名编译错误

现象：

- 在补“步骤表几何元数据有效性校验”时，直接手写这些历史乱码字段名，容易出现“当前上下文中不存在名称”“意外字符”等编译错误。

处理方式：

- `MainFrm.ManualSemiAuto.cs` 改成通过反射访问这些字段。
- 使用 ASCII 常量承接字段名，避免继续在逻辑代码里手写乱码字段。

### 5.9 中间线段丢失

现象：

- 某些步骤中，中间主折线会消失一段。

根因：

- 预览绘制仍按“步骤索引 = 线段索引”去跳过挤压段。

处理方式：

- 主折线完整绘制。
- 挤压作为独立图元叠加，不再通过“跳过主线”实现。

### 5.10 中间折弯左右方向不对

现象：

- 某些中间折弯方向与 `set` 布置图给出的方向不一致。

根因：

- 预览阶段板型仍在直接继承原图角度正负，而不是按当前步骤表里的 `折弯方向 + is色下` 推导。

处理方式：

- 引入 `BuildSemiAutoPreviewStageProfile()`。
- 预览专用阶段板型按当前步骤表推导预览角度符号。

## 6. 当前遗留问题

1. 个别复杂工单下，局部折弯方向和翻面后的左右关系仍需继续验证。
2. 预览页里还有少量未使用字段和残留变量需要清理。
3. 旧 `ini` 工单在重建步骤表后是否百分之百覆盖所有历史格式，仍需继续回归。

## 7. 建议的后续推进顺序

1. 先稳定 `view` 的复杂工单预览方向。
2. 再做 `view` 的小规模收尾清理：
   - 未使用字段
   - 无效变量
   - 局部残留按钮/控件
3. 最后再考虑是否把更多预览步骤诊断信息做成可视化输出，方便后续排查。

## 8. 当前结论

当前折弯预览功能已经从“入口混乱、职责混杂、状态各算各的”进入到“统一步骤表驱动”的阶段。

也就是说：

1. `set` 已经是唯一的折弯步骤编辑入口。
2. `view` 已经是唯一的折弯预览入口。
3. `set / manual / view` 已经共享 `CurtOrder.lstSemiAuto` 作为共同数据源。
4. 当前剩余问题，已经不再是“链路不清”，而是复杂工单下少数几类预览几何细节问题。
