# 折弯预览生成规则

## 1. 目的

本文档只定义 `折弯预览` 的生成规则，不记录调试现象，不讨论某一步“看起来对不对”。

预览画面必须由 `步骤表 + 折弯点设置 + 翻面规则` 正向生成，不能先生成一张图，再用经验去做 X/Y 镜像修补。

适用范围：

- `HMI/JSZW1000A/JSZW1000A/SubWindows/SubOPAutoView.cs`
- `HMI/JSZW1000A/JSZW1000A/SubWindows/SubOPAutoSet.cs`
- `HMI/JSZW1000A/JSZW1000A/MainFrm.cs`
- `HMI/JSZW1000A/JSZW1000A/MainFrm.SemiAuto.DerivedState.cs`

## 2. 字段语义

### 2.1 A/B 侧

- `A` 侧：从当前折弯点沿板型走向 `pxList[0]` 的那一支。
- `B` 侧：从当前折弯点沿板型走向 `pxList[^1]` 的那一支。
- `SubOPAutoSet` 画板中，`A` 和 `B` 标注是静态参考标识，不允许在预览里被几何结果反推覆盖。

### 2.2 内外选择

- `内外选择 = 0` 表示 `A-B(A在外侧)`。
- `内外选择 = 1` 表示 `B-A(B在外侧)`。

预览画面里的左右归属按以下口径解释：

- 当前步为 `A-B` 时：`B` 在设备左侧，`A` 在设备右侧成型。
- 当前步为 `B-A` 时：`A` 在设备左侧，`B` 在设备右侧成型。

这条规则优先级高于“主体在哪边更大/更长”的几何直觉。

### 2.3 折弯方向

- `折弯方向 = 0`：向上折弯
- `折弯方向 = 1`：向下折弯
- `is色下` 只表示做板开始时的放板正反面，是初始放料状态，不是翻面后的折弯方向开关。

`向上/向下` 只决定右侧成型区绕折弯点后的上下方向，不决定左右归属。

### 2.4 翻面

- 翻面可以是显式 `SemiAutoActionFlip`，也可以是相邻两步 `内外选择` 发生变化时的隐式翻面。
- 翻面含义是：板材在设备左侧基准内围绕触发翻面的折弯点旋转 180 度，不是“整板整体跑到画面右侧”。
- 翻面不改变设备内角的 `向上/向下折弯` 方向；翻面后的新折弯仍按步骤自身的折弯方向生成。
- 翻面只对翻面前已经折弯完成的板型做绕折弯点的 180 度旋转，不能把后续待折角的方向一起反向。
- 例：某个角在翻面前是“向下折弯”，翻面完成后，这个已经成形好的角在画面上应显示为绕折弯点旋转 180 度后的“向上”结果。
- 翻面完成后，下一步仍然必须重新按该步自己的 `A/B` 规则决定左右。

## 3. 预览生成总规则

### 3.1 左右规则

每一帧都必须先确定当前显示步的折弯点，再将板型切分成 `A` 支和 `B` 支。

然后只按当前步设置决定左右：

- 当前步是 `A-B`：左 = `B` 支，右 = `A` 支
- 当前步是 `B-A`：左 = `A` 支，右 = `B` 支

禁止使用以下错误规则：

- 哪边板材更长，就让哪边在左
- 哪边像主体，就让哪边在左
- 哪边平均 X 更小，就认定那边是左

### 3.2 上下规则

在左右归属确定后，右侧成型区的上下方向再由当前步的 `向上/向下折弯` 决定：

- 向上折弯：右侧成型区结果应在左侧参考边上方
- 向下折弯：右侧成型区结果应在左侧参考边下方

禁止使用以下错误规则：

- 用下一步的方向去修正当前已成型图形
- 用上一帧是否“更像”正确答案来决定要不要做 Y 轴镜像

### 3.3 翻面规则

翻面帧的生成规则是：

1. 先按上一帧结果得到当前已成型板型
2. 以触发翻面的折弯点为旋转中心，将当前已成型板型旋转 180 度
3. 旋转完成后，再按下一步的 `A/B` 规则重新布置左右

翻面只改变已成形板型在设备内的空间姿态，不允许反向后续步骤自身的 `向上/向下折弯` 方向。

## 4. DrawStep 规则

### 4.1 基本含义

- `DrawStep = 0`：初始待折状态
- 奇数步：当前待折角前一帧
- 偶数步：当前角执行完成后一帧

### 4.2 约束

- 奇数步显示的是“前几步已经完成后的板型”，不能再拿当前待折角的规则去二次镜像。
- 偶数步显示的是“当前角已执行完成后的板型”，应按当前角规则产生。
- 翻面完成态是单独一帧，不允许和普通奇偶步共用一套镜像补丁。

## 5. 挤压图元规则

- 挤压和开口挤压图元必须跟随该挤压动作发生时的 `is色下` 状态决定画在板材哪一侧。
- 挤压动作完成后如果发生翻面，挤压图元必须和主板型一起围绕触发翻面的折弯点旋转 180 度；例如向下挤压形成的角，翻面后画面上应成为向上的角。
- 挤压图元只负责表现当前状态，不得反向影响左右归属。
- 禁止把“挤压线画在上/下”的结果再拿来反推板型是否要整体镜像。

## 6. 代码落点

当前代码实现应收口到以下职责：

- `SubOPAutoSet`
  - 负责维护每一步的 `内外选择 / 折弯方向 / 回弹值 / 顺序`
- `MainFrm.SemiAuto.DerivedState`
  - 负责从步骤表重建 `is色下 / 后挡位置 / 操作提示`
- `SubOPAutoView`
  - 负责按当前步规则生成预览，不做经验性纠偏

## 7. 实现要求

实现时必须遵守：

1. 预览必须由规则正向生成。
2. 左右和上下是两条独立规则，不能混算。
3. 翻面必须是独立状态，不允许“顺手镜像一下”。
4. 禁止针对 `DrawStep 5/7/9` 之类个别步写死补丁。
5. 如果规则不够，先补规则，再改代码，不能先补代码再倒推规则。

## 8. SWI 逆向对照提取规则

从 `进口数控折弯机资料/SWI Engineering/SWI Folder Interface/FoldingMachineDll.dll` 的静态逆向结果看，SWI 折弯规划也遵守类似原则：

- 左右归属是显式状态，不是由几何长度、主体大小、平均 X 位置反推。
- 折弯方向和左右归属分离；方向参与成型上下关系和机械可行性校验，但不覆盖 A/B 左右。
- 翻面/镜像是独立状态迁移：先保留翻面前状态，再翻转方向标志并重新生成几何，不是画完以后做经验性镜像补丁。
- 每个候选结果都保留“动作状态 + 几何状态”的成对快照，后续预览应从状态快照正向生成。
- 颜色侧、挤压、压边、Squash/Swage 一类图元是当前状态的表现，不得反向决定板型左右或是否整体镜像。
- 几何校验失败时，只能通过明确的翻面/镜像状态迁移重试，不能在渲染层临时补 X/Y 镜像。

因此本项目预览实现应额外遵守：

1. 每一帧预览必须先得到明确的 `步骤状态快照`，再画图。
2. `A/B 左右归属`、`向上/向下`、`is色下`、`翻面状态` 必须是彼此独立的输入项。
3. 翻面帧只能修改已完成折弯角的镜像关系，不能修改下一步自身的折弯方向。
4. 渲染结果不能作为规则来源；如果发现图形不对，应回查步骤状态、A/B 归属、翻面状态和方向状态。
5. 调试时应尽量记录当前帧的状态码或派生状态，避免靠“看起来像”判断是否需要镜像。

## 9. 预览动画规则

### 9.1 基本原则

- 动画只能表现两个已定义状态之间的过渡，不能成为生成板型规则的来源。
- 每段动画必须先确定 `起始状态快照` 和 `结束状态快照`，再做插值。
- 动画结束帧必须和对应的静态 `DrawStep` 结果完全一致。
- 禁止为了让动画中间帧“看起来顺”而修改 A/B 左右、翻面或折弯方向规则。

### 9.2 折弯动画

折弯动画用于表现“当前角从未折到折完”的过程。

生成规则：

1. 起始状态取当前角执行前的板型。
2. 结束状态取当前角执行完成后的板型。
3. 固定设备左侧参考支，围绕当前折弯点旋转右侧成型支。
4. 旋转方向只由当前步 `向上/向下折弯` 决定。
5. 左右归属仍按当前步 `A/B` 规则决定，不允许按旋转后的位置重新判断左右。

动画插值只允许作用于当前正在折弯的那一支：

- 已经完成的历史折弯保持起始状态中的成型结果。
- 尚未执行的后续折弯不得提前反向或镜像。
- 当前折弯角从 `0` 插值到目标角度，回弹值只参与目标角度，不作为额外镜像规则。

### 9.3 翻面动画

翻面动画用于表现板材在设备左侧基准内的镜像过程。

生成规则：

1. 起始状态取翻面前已经完成的板型。
2. 结束状态取翻面后的独立状态帧。
3. 镜像只作用于翻面前已经完成的折弯角。
4. 翻面动画不改变下一步自身的 `向上/向下折弯` 方向。
5. 翻面完成后，下一步必须重新按该步的 `A/B` 规则布置左右。

禁止事项：

- 禁止把整板简单平移到画面另一侧当作翻面。
- 禁止把翻面动画和普通折弯动画合并成同一段插值。
- 禁止在翻面动画里提前执行下一步折弯方向。

### 9.4 DrawStep 与动画段关系

- `DrawStep = 0` 只能作为初始静态帧，不做折弯插值。
- 奇数步作为当前折弯动画的起始参考帧。
- 偶数步作为当前折弯动画的结束参考帧。
- 翻面完成态必须作为独立动画结束帧，不能复用普通奇偶步镜像逻辑。

一段动画的端点关系必须满足：

- 折弯动画：`当前角前一帧 -> 当前角完成帧`
- 翻面动画：`翻面前完成帧 -> 翻面完成帧`
- 翻面后的第一折：`翻面完成帧 -> 下一折完成帧`

### 9.5 图元和提示动画

- 挤压、开口挤压、颜色侧图元跟随当前状态快照显示。
- 这些图元可以渐显、移动或切换，但不能反向决定板型是否镜像。
- 后挡位置、操作提示、当前折弯点高亮应跟随 `MainFrm.SemiAuto.DerivedState` 的派生状态。
- 如果动画中间帧需要显示提示，应显示当前动画段的起始/执行中状态，不得提前显示下一步完成后的提示。

### 9.6 动画验证要求

每段动画至少验证：

1. 起始帧是否等于对应静态起始 `DrawStep`。
2. 结束帧是否等于对应静态结束 `DrawStep`。
3. 中间帧是否只插值当前动作，不改变 A/B 归属。
4. 翻面中间帧是否只表现设备左侧基准内镜像。
5. 翻面后第一折是否重新按下一步规则决定左右和上下。

## 10. 验证清单

每次修改后至少验证：

1. 初始态 `DrawStep 0`
2. 第一个角完成态
3. 第一个翻面帧
4. 第一个翻面后的第一折
5. 最终完成态

并检查以下 4 点是否同时成立：

- 左右是否符合当前步的 `A/B` 规则
- 上下是否符合当前步的 `向上/向下折弯`
- 翻面是否只发生在设备左侧基准内
- 挤压图元是否跟随当前 `is色下`
