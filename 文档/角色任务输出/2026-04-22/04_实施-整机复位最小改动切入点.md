# 04_实施-整机复位最小改动切入点

## 1. 实施总判断

如果后续进入代码阶段，首批不要先动 HMI。当前最小可评审切入点应放在 PLC：

- HMI 入口已经存在：`SubOPManual -> Hmi_bArray[41]`
- PLC 总入口已经存在：`fbMachine -> bReset / rtReset / iWholeRst`
- 真正缺口在于：`iWholeRst` 为空、分条回原位未任务化

## 2. 建议批次

### 批次 1A：先补分条机回原位任务化

建议文件：

- `26.4/Tc_JSZW400/JSZW/Untitled1/DUTs/DUTs/E_UnitTask.TcDUT`
- `26.4/Tc_JSZW400/JSZW/Untitled1/POUs/UNIT/fb_Slitter/FB_SlitterAct.TcPOU`
- `26.4/Tc_JSZW400/JSZW/Untitled1/POUs/UNIT/fb_Slitter/fb_Slitter.TcPOU`

目的：

- 增加 `eTask_SlitterHome`
- 让分条回原位从 HMI 位直驱转为 PLC 内部任务

### 批次 1B：再只改 `fbMachine.TcPOU`

建议文件：

- `26.4/Tc_JSZW400/JSZW/Untitled1/POUs/fbMachine.TcPOU`

目的：

- 在 `CASE iWholeRst OF` 中编排整机复位顺序
- 只复用现有 `TaskClamp / TaskBackGauge / TaskApron / TaskTable / TaskFeed / TaskSlitter`

## 3. 首批明确先不动的文件

- `HMI/JSZW1000A/JSZW1000A/SubWindows/SubOPManual.cs`
- `HMI/JSZW1000A/JSZW1000A/SubWindows/SubOPManual.Designer.cs`
- `HMI/JSZW1000A/JSZW1000A/MainFrm.cs`
- `26.4/Tc_JSZW400/JSZW/Untitled1/POUs/FB_Generic.TcPOU`
- `26.4/Tc_JSZW400/JSZW/Untitled1/DUTs/DUTs/ST_FolderCom.TcDUT`
- `26.4/Tc_JSZW400/JSZW/Untitled1/Untitled1.plcproj`

原因：

- HMI 入口已够用，先不扩大改动面。
- `FB_Generic` 一改就是 PackML 基类级影响，风险过大。
- 当前不新增文件时，也不需要先改工程清单。

## 4. 预期风险

1. `rtReset.Q` 是脉冲，`iWholeRst` 必须内部自保持。
2. 分条不任务化的话，整机复位状态机容易被迫写 HMI 位。
3. 翻板复位必须按工作侧和分条互锁顺序执行，不能并发乱发任务。

## 5. 最快验证方式

PLC 首批只做静态检查：

```powershell
[xml](Get-Content -LiteralPath "26.4/Tc_JSZW400/JSZW/Untitled1/POUs/fbMachine.TcPOU" -Raw) | Out-Null
```

批次 1A 完成后补查关键词：

```powershell
rg -n "eTask_SlitterHome|TaskSlitter|CmdHome|Hmi_bArray\\[44\\]" "26.4/Tc_JSZW400/JSZW/Untitled1"
```

## 6. 关键结论

1. 首批应从 PLC 切，不应从 HMI 切。
2. 先补分条回原位任务化，再填 `iWholeRst`，是最小、最稳的顺序。
