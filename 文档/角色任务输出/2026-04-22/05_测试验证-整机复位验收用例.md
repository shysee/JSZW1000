# 05_测试验证-整机复位验收用例

## 1. 验证拆分

本任务分成两层：

- 离线检查：只验证代码结构、状态位、任务位和入口路径
- 现场验证：验证真实机械回位、安全性、急停/恢复和报警场景

## 2. 离线已确认项

1. `Hmi_bArray[41]` 是当前可见整机复位主入口。
2. `fbMachine` 中 `rtReset.Q` 会清空 `TaskClamp / TaskBackGauge / TaskApron / TaskSlitter / TaskTable / TaskFeed`。
3. `FB_Generic.M_Resetting()` 当前仅 `M_StateComplete()`，不能把“进入 Resetting/退出 Resetting”直接当作整机已安全复位。
4. `GVL_IO.In_btnReset` 已映射，但当前源码未见直接消费路径，必须现场确认。
5. 当前代码能明确覆盖的工位有：夹钳、后挡、台面、送料、翻板、分条；出料平台未见独立任务通道。

## 3. 推荐静态检查命令

```powershell
$root='26.4/Tc_JSZW400/JSZW'
$files=Get-ChildItem -Path $root -Recurse -File -Include *.TcPOU,*.TcDUT,*.TcGVL,*.plcproj,*.tsproj
$bad=@()
foreach($f in $files){
  try { [xml](Get-Content -LiteralPath $f.FullName -Raw) | Out-Null }
  catch { $bad += $f.FullName }
}
if($bad.Count -eq 0){ "XML_OK $($files.Count)" } else { "XML_FAIL $($bad.Count)"; $bad }
```

本轮实跑结果：`XML_OK 104`

说明：这只是 XML 静态检查，不是 TwinCAT 完整编译。

## 4. 单工位现场用例

1. 夹钳复位：确认 `Clamp_AtZeroPoint` 和互锁一致。
2. 后挡复位：确认 `BackGauge_AtHome`。
3. 台面/送料复位：确认 `Table_AtHome / Feed_AtHome`。
4. 翻板复位：确认上下翻板折弯原点和滑台安全位。
5. 分条复位：确认 `Slitter_AtHome` 且互锁有效。

## 5. 整机现场用例

1. 正常整机复位：所有工位从非初始位回到初始位。
2. 带残留任务整机复位：已有任务执行中触发复位，确认先清任务再回位。
3. 连续重复复位：确认幂等，不抖动、不堆任务。

## 6. 急停/暂停/恢复用例

1. 整机复位中触发急停。
2. 急停解除后重新发起复位。
3. `Hold/Unhold`
4. `Suspend/Unsuspend`

重点检查：

- 不得自动续跑危险动作
- 恢复路径必须可解释

## 7. 失败回退/报警确认用例

1. 原点信号异常
2. 互锁不满足
3. 报警未解除时发起复位
4. HMI 复位与物理复位按钮一致性确认
5. 现场附加机构未纳入代码闭环时的单独登记

## 8. 关键结论

1. 当前结构足以编写验收方案，但不足以离线证明整机复位已真正落地。
2. 出料平台和物理复位按钮是两项必须现场确认的边界。
