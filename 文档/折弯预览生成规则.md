# 折弯预览生成规则

## 1. 目的与范围

本文档定义 `折弯预览` 的目标规则、当前实现映射、当前 checkout 观察边界，以及后续测试口径。

本文档解决的是 4 个问题：

1. 预览图应该由哪些输入状态正向生成
2. `DrawStep / FLIP / REGRIP / 后挡送料 / 折弯` 之间的时间线怎样解释
3. 当前代码分别在哪些方法里承担这些职责
4. 后续改代码和补测试时，什么可以动，什么不能靠“看起来像”去补

本文档不记录零散调试现象，也不为某个 `DrawStep 5/7/9` 之类个别步编造例外补丁。

适用范围：

- `HMI/JSZW1000A/JSZW1000A/MainFrm.cs`
- `HMI/JSZW1000A/JSZW1000A/MainFrm.SemiAuto.cs`
- `HMI/JSZW1000A/JSZW1000A/MainFrm.SemiAuto.DerivedState.cs`
- `HMI/JSZW1000A/JSZW1000A/SubWindows/SubOPAutoSet.cs`
- `HMI/JSZW1000A/JSZW1000A/SubWindows/SubOPAutoView.cs`
- `HMI/JSZW1000A/JSZW1000A/SubWindows/SubOPAutoView.PreviewTimeline.cs`
- `HMI/JSZW1000A/JSZW1000A/SubWindows/SubOPAutoView.PreviewText.cs`

## 2. 当前生效口径（优先级最高）

以下口径覆盖旧规则；若后文和本节冲突，以本节为准。

1. `右侧活动端` 是固定规则。真正折弯时，旋转的是画面右侧成型支，不再根据长度、外形、后挡位置去猜哪一边是活动端。
2. `A-B / B-A` 只决定 `左右归属` 与 `是否触发 FLIP`，不直接决定画面里的 `上折 / 下折`。
3. 自动生成方案的 `step.折弯方向` 必须先在生成/映射层归一为正式值；预览、左栏文案、保存、发送都应读取这一份正式字段。
4. 自动生成方案的方向真值来源只认 `长角序号 -> CurtOrder.lengAngle[idx].Angle`。没有有效源角映射时，不允许再从预览几何结果反推方向符号。
5. `FLIP` 是独立过渡段。真正折弯步围绕 `坐标序号` 对应的折弯支点旋转；`FLIP` 过渡围绕机床预览参考中线旋转，不复用上一折的支点。
6. `后挡位置` 不改写正式折弯支点，也不作为预览碰撞区放置依据。送料态预览直接把当前折弯点放到机床折弯原点，用该对齐姿态做碰撞比较。当前时间线口径为：`FLIP -> REGRIP -> 后挡送料 -> 折弯`。
7. `送料 / 待折态` 的方向口径跟下一步折弯动作一致，不单独发明一套送料方向。送料态的方向文案、工作翻板选择、工作翻板碰撞区比较，都应读取“下一帧折弯完成态会使用的方向”。

## 3. 状态模型

### 3.1 原始输入状态

折弯预览只能依赖以下显式输入，不允许从最终图形倒推：

- `板型输入`
  - `CurtOrder.lengAngle`
  - `CurtOrder.pxList`
  - `CurtOrder.Width`
- `步骤输入`
  - `行动类型`
  - `折弯序号`
  - `长角序号`
  - `坐标序号`
  - `内外选择`
  - `折弯方向`
  - `折弯角度`
  - `后挡位置`
  - `回弹值`
  - `抓取类型`
  - `重新抓取`
- `初始放料输入`
  - `CurtOrder.st逆序`
  - `CurtOrder.st色下`

### 3.2 字段语义

#### 3.2.1 A/B 侧

- `A` 侧：从当前折弯点沿板型走向 `pxList[0]` 的那一支
- `B` 侧：从当前折弯点沿板型走向 `pxList[^1]` 的那一支
- `SubOPAutoSet` 里的 `A/B` 标注是静态参考标识，不允许在预览阶段被几何结果反推覆盖

#### 3.2.2 内外选择

- `内外选择 = 0` 表示 `A-B (A 在外侧)`
- `内外选择 = 1` 表示 `B-A (B 在外侧)`

在预览里，左右归属固定解释为：

- 当前步是 `A-B`：左 = `B` 支，右 = `A` 支
- 当前步是 `B-A`：左 = `A` 支，右 = `B` 支

这条规则优先级高于“哪边更长 / 更像主体 / 平均 X 更小”之类几何直觉。

#### 3.2.3 折弯方向

- `折弯方向 = 0`：向上折弯
- `折弯方向 = 1`：向下折弯
- `is色下` 只表示初始放料正反面，不是翻面后的折弯方向开关

`向上 / 向下` 只决定右侧成型区绕折弯点后的上下方向，不决定左右归属。

#### 3.2.4 翻面

翻面可以来自两种来源：

- 显式 `SemiAutoActionFlip`
- 相邻两步 `内外选择` 变化导致的隐式翻面

翻面的语义是：

- 板材在设备左侧基准内完成 `180°` 旋转
- 只改变已经完成折弯部分的空间姿态
- 不改写下一步自身的 `折弯方向`
- 翻面完成后，下一步仍重新按该步自己的 `A/B` 规则决定左右

### 3.3 派生状态

预览层允许使用的派生状态只有以下几类：

- `当前显示步`
- `当前已应用步数`
- `当前 DrawStep`
- `当前是否处于 FLIP 完成态`
- `当前是否处于送料态 / 折弯完成态`
- `当前 is色下`
- `当前提示文案`
- `当前右侧成型区碰撞判定结果`

这些派生状态必须由步骤表正向计算得到，不能由渲染结果倒推。

## 4. 静态预览生成规则

### 4.1 总原则

每一帧静态预览都必须按以下顺序生成：

1. 确定当前显示步和当前 `DrawStep`
2. 确定该帧对应的已应用步骤数
3. 从平板或阶段板型重建当前板型快照
4. 应用当前步的 `A/B` 左右规则
5. 应用当前步的 `向上/向下` 上下规则
6. 如当前帧是翻面完成态，则单独应用翻面规则
7. 再做碰撞判定、提示文案和颜色/挤压图元更新

禁止反过来做：

- 先画出一张图，再决定左右
- 先看图形朝上朝下，再修正 `折弯方向`
- 先用镜像让画面“看起来对”，再补规则解释

### 4.2 左右规则

每一帧都必须先确定当前显示步的折弯点，再将板型切成 `A` 支和 `B` 支。

然后只按当前步的 `内外选择` 决定左右：

- `A-B`：左 = `B`，右 = `A`
- `B-A`：左 = `A`，右 = `B`

禁止使用以下错误规则：

- 哪边板材更长，就让哪边在左
- 哪边更像主体，就让哪边在左
- 哪边平均 X 更小，就把哪边认成左

### 4.3 上下规则

在左右归属确定后，右侧成型区的上下方向再由当前步的 `折弯方向` 决定：

- 向上折弯：右侧成型区结果应出现在左侧参考边上方
- 向下折弯：右侧成型区结果应出现在左侧参考边下方

禁止使用以下错误规则：

- 用下一步的方向修正当前已成型图形
- 用上一帧“更像正确答案”的结果去决定是否做 Y 镜像

### 4.4 翻面规则

翻面帧的生成规则是：

1. 先取得翻面前已成型板型
2. 以翻面参考基准为旋转中心，将当前已成型板型旋转 `180°`
3. 旋转完成后，再按下一步 `A/B` 规则重新布置左右

翻面只改变已成形板型的空间姿态，不允许：

- 反向修改后续步骤的 `向上 / 向下`
- 把整板平移到另一边代替翻面
- 把翻面和普通折弯揉成同一套镜像补丁

### 4.5 DrawStep 规则

#### 4.5.1 基本含义

- `DrawStep = 0`：初始待折状态
- 奇数步：当前待折角之前的送料/待折状态
- 偶数步：当前角执行完成后的折弯完成状态
- `showingFlipCompletionState = true`：翻面完成态，独立于普通奇偶步

#### 4.5.2 约束

- 奇数步显示的是“前几步已经完成后的板型”，不能再拿当前待折角规则去二次镜像
- 偶数步显示的是“当前角已执行完成后的板型”，应按当前角规则产生
- 翻面完成态必须是单独一帧，不允许和普通奇偶步共用一套镜像补丁

## 5. 动画与时间线规则

### 5.1 时间线口径

当前目标时间线解释为：

`FLIP -> REGRIP -> 后挡送料 -> 折弯`

其中：

- `FLIP`：姿态翻转
- `REGRIP`：夹持/抓取条件切换
- `后挡送料`：显示下一折支点已经对齐机床折弯原点后的待折姿态
- `折弯`：围绕当前支点旋转右侧成型支

### 5.2 动画总原则

动画只能表现两个已定义状态之间的过渡，不能成为状态来源。

每段动画必须先确定：

- `起始状态快照`
- `结束状态快照`

再做插值。动画结束帧必须与对应静态 `DrawStep` 完全一致。

禁止为了让中间帧“看起来顺”而修改：

- `A/B` 左右归属
- `折弯方向`
- `FLIP` 是否发生
- `is色下`

### 5.3 折弯动画

折弯动画用于表现“当前角从未折到折完”。

生成规则：

1. 起始状态 = 当前角执行前的板型
2. 结束状态 = 当前角执行完成后的板型
3. 固定左侧参考支
4. 围绕当前折弯点旋转右侧成型支
5. 旋转方向只由当前步 `折弯方向` 决定

限制：

- 已完成历史折弯保持不变
- 未执行的后续折弯不得提前翻转
- 回弹值只参与目标角度，不参与镜像规则

### 5.4 翻面动画

翻面动画用于表现设备左侧基准内的翻面过程。

生成规则：

1. 起始状态 = 翻面前已完成板型
2. 结束状态 = 翻面完成帧
3. 只作用于翻面前已完成的折弯角
4. 不改变下一步自身 `折弯方向`
5. 翻面后第一折重新按下一步规则布置左右和上下

禁止事项：

- 把整板简单平移到另一边当作翻面
- 把翻面动画和普通折弯动画合并
- 在翻面动画里提前执行下一步折弯方向

### 5.5 DrawStep 与动画段关系

- `DrawStep = 0`：初始静态帧
- 奇数步：送料/待折起点
- 偶数步：折弯完成终点
- `showingFlipCompletionState`：翻面完成终点

端点关系必须满足：

- 折弯动画：`当前角前一帧 -> 当前角完成帧`
- 翻面动画：`翻面前完成帧 -> 翻面完成帧`
- 翻面后第一折：`翻面完成帧 -> 下一折完成帧`

## 6. 图元、提示与颜色侧规则

### 6.1 挤压图元

- 挤压和开口挤压图元必须跟随动作发生时的 `is色下`
- 动作完成后若发生翻面，挤压图元必须和主板型一起翻转
- 挤压图元只表现状态，不得反向影响左右归属

### 6.2 颜色侧

- 颜色线只表现当前 `is色下`
- `is色下` 的切换只能来自初始化状态或 `FLIP`
- 不允许根据颜色线位置反推当前板型是否需要镜像

### 6.3 提示文案

提示文案必须基于当前状态快照生成，不允许超前显示下一步结果。

当前至少需要覆盖：

- 后挡送料提示
- `FLIP` 提示
- `FLIP` 完成提示
- 折弯方向提示
- 挤压提示

## 7. 碰撞区与红线规则

### 7.1 目的

碰撞区规则只服务于预览中的硬碰撞标记，用于：

- 让危险线段与设备结构一致
- 区分送料完成态和折弯完成态的判定对象
- 取代“右侧大矩形包络”之类经验性粗判定

本节不替代 PLC 安全互锁，也不把翻板扫掠扇形当作主碰撞依据。

### 7.2 红线语义

- 红线只表示 `硬碰撞`
- 红线只在 `状态落点` 判断，不在动画中间帧实时跳变
- 红线不表示 `近碰撞 / 需复核`

### 7.3 判定对象

当前预览碰撞判定对象是：**当前步右侧成型图形**

说明：

- 左侧参考支不参与当前步钳口硬碰撞判定
- 如果某个折弯点对应的右侧成型图形违规，就把该折弯点相连的两根线段标红

### 7.4 设备结构口径

当前按 SWI/现场设备结构统一理解：

- 中间是固定钳口
- 上下各有一块可动翻板
- 翻板不工作时贴各自钳口外侧回退停放
- 翻板工作时前缘移动到折弯点位置，翻板近端水平边的左端点与折弯点重合
- 钳口、送料区、上钳口折弯边界、下折弯边界都是固定机器结构区，不随步骤、后挡位置或折弯完成状态缩放
- 只有上下翻板的工作/停靠位置随当前显示步切换

### 7.5 当前已确认参数口径

- `夹臂宽度 = 12 mm`
- `压钳臂顶角度 = 45°`
- `上钳口厚度 = 15 mm`
- `翻板高度 / 宽度 = 15 mm`
- `压钳臂底角度 = 35°`
- `翻板占位长度 = 400 mm`
- `翻板斜率角 = 80°`
- `停放翻板回退高度 = 50 mm * 回退等级`
- `轻微/严重碰撞分界比例 = 30%`

这些值必须作为碰撞区参数输入，不能写死成某个案例的编号区域。后续现场修正时，优先调整这些参数或边界线定义，而不是改红区编号。

### 7.6 碰撞区数学化口径

以当前显示步的折弯点为坐标原点 `O = (0, 0)`。所有碰撞区边界都表示为有方向直线 `L(a, b)`，并用“画面左侧半平面”做区域划分：

```text
side(P, L) = cross(b - a, P - a)
P 在 L 的画面左侧 <=> side(P, L) >= -eps
P 在 L 的画面右侧 <=> side(P, L) <= eps
```

边界接触算碰撞，因此半平面判断必须包含等号。`eps` 只用于浮点容差，不是安全余量。

当前参与划分的 5 条边界线：

- `upperClamp`：上钳口折弯边界线
- `lowerClamp`：下钳口折弯边界线
- `upperFlap`：上翻板外沿线，按当前步工作/停靠位置计算
- `lowerFlap`：下翻板外沿线，按当前步工作/停靠位置计算
- `horizontalTangent`：过折弯点的水平切线 `y = 0`，用于把上下翻板区域切开

`horizontalTangent` 当前只参与调试分区和显示，不单独决定红区命中。真正红色碰撞区由工作翻板状态决定：

```text
下翻板工作位：
  danger = left(lowerFlap) AND (left(upperClamp) OR left(upperFlap))

上翻板工作位：
  danger = left(upperFlap) AND (left(lowerClamp) OR left(lowerFlap))
```

解释：

- 工作翻板外沿线是第一道必须越过的边界
- 对侧钳口折弯边界和对侧停靠翻板外沿线共同决定另一侧危险区
- 蓝色编号区只是由 5 条线切出来的候选分区，只有满足上述公式的区域才是红色碰撞区
- 不能再用固定编号 `5/7/8/9/10` 作为正式规则；编号只用于人工确认当前参数下的显示结果

### 7.7 送料完成态判定

送料完成态用于检查后挡定位后的板型是否能安全进入当前折弯点。送料区是固定机器区：在折弯点左侧，由水平面 `y = 0` 与上钳口折弯边界围成。它不是随板型或后挡位置重算出来的包络。

预览实现约束：

- 送料态不再根据 `后挡位置` 计算值做画面平移
- 送料态不再在上一折参考坐标和当前折弯点坐标之间插值
- 当前显示步的折弯点必须直接落在机床折弯原点，再进行送料区、钳口区和翻板区比较
- 送料态的工作翻板方向必须跟下一步折弯动作方向一致，不能继续沿用上一折或当前画面残余几何方向

判定为硬碰撞的条件：

- 右侧成型折弯点落入上钳口判定区
- 右侧成型折弯点落入当前显示折弯步对应的工作翻板区
- 右侧成型折弯点落入停靠翻板区
- 右侧成型自由端的第 1 / 2 端点进入上述判定区，也算硬碰撞

### 7.8 折弯完成态判定

折弯完成态只判断折后板型是否进入显式碰撞区。固定碰撞区大小和送料完成态一致，不能因为当前帧是折弯完成态就重新缩放或移动。

主判定：

- 右侧成型折弯点进入上钳口判定区
- 右侧成型折弯点进入工作翻板区
- 右侧成型折弯点进入停靠翻板区
- 右侧成型自由端的第 1 / 2 端点进入上述判定区，也算硬碰撞
- 右侧成型折弯点或端点越过下钳口折弯边界，也算硬碰撞

### 7.9 翻板碰撞区

翻板碰撞区是活动区，但几何尺寸固定：

- 工作翻板和停靠翻板都绘制为平行四边形
- 平行四边形近端边必须保持水平
- 工作翻板近端水平边的左端点必须与折弯点重合
- 上翻板和下翻板使用同一条端点规则，只是延伸方向不同
- 送料完成态和折弯完成态都应显示翻板碰撞区，便于核对当前活动件位置
- 非 FLIP 帧都应把当前工作翻板纳入红线硬碰撞判定；FLIP 完成态不做工作翻板硬碰撞判定
- 碰撞判定包含“区域内部”和“边界接触”；压在线上的边界点也算碰撞

上下折弯对应的工作翻板规则：

- `折弯方向 = 0`（向上折弯）：下翻板处于工作位，上翻板按停靠位显示
- `折弯方向 = 1`（向下折弯）：上翻板处于工作位，下翻板按停靠位显示
- 不能用板型最终朝向或画面红线结果反推工作翻板，只能读取当前显示步的正式 `折弯方向`

### 7.10 红线粒度

红线判定以当前帧板型点列为输入，判定主体是点列中的已成型折弯点，红线输出主体是线段。

规则：

1. 先确定当前折弯点 `anchorIndex`
2. 只遍历当前步右侧成型图形中的点
3. 当前折弯点本身不做碰撞点判定
4. 若某个已成型折弯点进入固定钳口区、送料区边界外或当前翻板判定区，则该点为碰撞点
5. 碰撞点只把它直接相邻的两条线段加入红线集合：`pointIndex - 1` 和 `pointIndex`
6. 红线不向更远线段扩散
7. 当前折弯点与相邻点形成的当前折弯工作边不加入红线集合

当前折弯工作边是当前折弯要求下必然允许的线段。风险来自再前一个点及更早的已成型折弯点。

### 7.11 碰撞严重度

碰撞严重度在“已确认命中红色碰撞区”之后计算，不改变碰撞区本身。

对每条被碰撞点牵连的相邻线段，计算该线段落入红色碰撞区的长度比例：

```text
collisionRatio = length(segment ∩ dangerZone) / length(segment)
```

显示规则：

- `collisionRatio = 0` 且只是边界接触：轻微碰撞，黄色
- `0 < collisionRatio <= 30%`：轻微碰撞，黄色
- `collisionRatio > 30%`：严重碰撞，红色

`30%` 是当前验证阈值，应作为参数保留。后续可按现场安全余量调整，不应写死到几何公式里。

当前不采用：

- 整条轮廓整体标红
- 只截局部一点染红
- 连续采样命中后整段轮廓都染红
- 单纯以 `y` 的上下限当作主碰撞规则
- 通过镜像判定点把右侧点翻到左侧再判断

## 8. 代码实现映射

### 8.1 `MainFrm.cs`

- 不允许在页面切换时额外偷偷改写某一步的预览方向

### 8.2 `MainFrm.SemiAuto.cs`

实现要求：

### 8.3 `MainFrm.SemiAuto.DerivedState.cs`

- `ShouldApplyPreviewFlipAfterStep(...)`
  - 负责决定某一步后是否要应用预览层的翻面状态迁移

当前方法级口径：

- `BuildSemiAutoPreviewStageProfile(...)` 先从平板开始重放步骤

- `RotateWholePreviewProfile(...)` 负责对翻面步做整体验证口径下的 `180°` 旋转

实现要求：

### 8.5 `SubOPAutoView.PreviewTimeline.cs`

职责：

- 负责 `DrawStep` 时间线推进
- 负责把阶段板型转成当前显示帧
- 负责送料态、折弯态、翻面完成态之间的切换

当前关键方法：

- `PreViewSt()`
  - 时间线推进总入口
  - 先判断是否要插入翻面完成态
  - 再推进 `iDrawStep`
- `GetCurrentPreviewStepIndex()`
  - 从 `iDrawStep` 反推当前步骤索引
- `ShouldFlipAfterCurrentPreviewStep(...)`
  - 判断偶数折弯完成态后是否需要单独插入翻面完成帧
- `BuildPreviewSegmentsForDrawStep(int drawStep)`
  - 这是当前 checkout 里最重要的“帧组装”方法
  - 决定当前显示步
  - 决定当前已应用步数
  - 决定当前 `is色下`
  - 读取阶段板型
  - 把阶段板型转到当前步的显示坐标系
  - 在送料态做前后帧插值
  - 在翻面完成态做独立旋转表现
- `RecalculatePreviewCollisionSegmentsForCurrentDraw()`
  - 基于当前帧轮廓重算红线段

当前方法级解释：

1. `BuildSemiAutoPreviewStageProfile(...)`
   - 先产生“机床坐标系下”的阶段板型
2. `TransformPreviewProfileToStepFrame(...)`

3. `InterpolatePreviewProfiles(...)`
   - 当前送料态做中间过渡
4. `RotatePreviewProfileAroundScreenCenter(...)`
   - 当前 checkout 里翻面完成态在表现层做独立旋转展示

这一层最容易写歪，约束如下：

- 可以做插值
- 可以做屏幕坐标变换
- 不允许把插值结果反写成规则来源

## 9. 当前 checkout 的实现边界

本节是“按当前 checkout 观察”的实现边界，不是永久规范。后续代码继续调整时，这一节允许更新。

### 9.1 当前已经基本成形的点

1. 预览正式方向的归一化已经集中到 `NormalizeSemiAutoStepsForPreview(...)`
2. 阶段板型重放已经集中到 `BuildSemiAutoPreviewStageProfile(...)`
3. 当前帧颜色侧推导已经集中到 `ResolveSemiAutoPreviewColorDown(...)`
4. `DrawStep` 到“当前步 / 已应用步数 / 翻面完成态”的换算已经集中到 `BuildPreviewSegmentsForDrawStep(...)`
5. 碰撞红线已经从“经验性大包络”转向参数化边界 + 右侧成型区判定

### 9.2 当前仍应视为易变区的点

1. `TransformPreviewProfileToStepFrame(...)`
   - 当前既承担“对齐左侧参考支”的工作，也隐含了显示坐标系选择
   - 后续如果要拆状态层和表现层，这里大概率还会继续变化
2. `RotatePreviewProfileAroundScreenCenter(...)`
   - 当前 checkout 的翻面完成态表现仍偏 UI 层
   - 若后续把翻面基准彻底下沉到状态层，这里可能收缩
3. 送料态的插值方式
   - 当前 `InterpolatePreviewProfiles(..., 0.5)` 仍是表现层折中，不是严格物理送料动画
4. `FLIP -> REGRIP -> 后挡送料 -> 折弯` 的细粒度显示切分
   - 规则口径已经定下，但实际帧级拆分仍可能继续演进

### 9.3 当前明确不应该回退的点

1. 不要回到“先画出来，再靠 X/Y 镜像修”
2. 不要在 `SubOPAutoView` 里重新猜正式方向
3. 不要按个别 `DrawStep` 写死补丁
4. 不要让颜色线、挤压线、碰撞红线反向决定板型左右和翻面

## 10. 测试与验收清单

### 10.1 静态落点最少验证

每次涉及预览规则修改后，至少验证：

1. `DrawStep 0` 初始态
2. 第一个角完成态
3. 第一个翻面完成帧
4. 第一个翻面后的第一折
5. 最终完成态

并同时检查：

- 左右是否符合当前步 `A/B`
- 上下是否符合当前步 `折弯方向`
- 翻面是否只作用于已完成折弯部分
- 颜色线和挤压图元是否跟随当前 `is色下`

### 10.2 动画最少验证

每段动画至少验证：

1. 起始帧是否等于对应静态起点
2. 结束帧是否等于对应静态终点
3. 中间帧是否只插值当前动作，不改 `A/B`
4. 翻面中间帧是否只表现翻面，不提前执行下一折方向
5. 翻面后第一折是否重新按下一步规则决定左右和上下

### 10.3 碰撞最少验证

至少验证两个阶段：

1. 送料完成态
   - 是否切换到当前显示折弯步的工作翻板区比较
2. 折弯完成态
   - 是否只按边界角域做主判定

同时验证：

- 红线是否只标当前违规折弯点相连两段
- 不会整轮廓误染红

### 10.4 建议优先补的自动化测试

优先放在 `HMI/JSZW1000A/JSZW1000A.Tests/SemiAutoPreviewTests.cs`：

1. `NormalizeSemiAutoStepsForPreview(...)`
   - 自动生成步骤方向归一化
   - `is色下` 沿步骤传播
2. `ResolveSemiAutoPreviewColorDown(...)`
   - 显式翻面
   - 隐式翻面
   - `applyFlipAfterLastIncludedStep` 分支
3. `BuildSemiAutoPreviewStageProfile(...)`
   - 普通折弯
   - 翻面前后
   - 首尾挤压不污染正式折弯方向
4. `BuildPreviewSegmentsForDrawStep(...)`
   - `DrawStep 0 / 奇数 / 偶数 / showingFlipCompletionState`
   - 翻面完成帧是否切到下一可渲染步
5. 预留/兼容类测试
   - 手工步骤表 `custom-manual`
   - 老方案字段解析
   - `后挡位置` 与元数据保留

## 11. 实施约束

后续任何折弯预览改动都必须遵守：

1. 先改规则，再改代码
2. 正式方向只在生成/归一化层确定
3. 预览状态重放和屏幕表现尽量分层
4. 不能用渲染结果倒推规则
5. 不能为了过一个个案而写死 `DrawStep` 特判补丁
