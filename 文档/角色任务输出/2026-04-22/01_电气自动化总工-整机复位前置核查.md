# 01_电气自动化总工-整机复位前置核查

## 1. 目标范围

本轮只做“设备各工位复位至初始状态”的前置核查，不改 PLC/HMI 代码。重点确认：

- 现有代码里哪些工位已具备回初始位的状态与任务基础。
- 整机复位建议顺序是否能复用现有任务总线。
- 哪些互锁是代码已确认，哪些必须现场确认。

## 2. 代码中已确认的工位与初始状态口径

### 2.1 后挡/台面/送料

文件：

- `26.4/Tc_JSZW400/JSZW/Untitled1/POUs/UNIT/fb_BackGauge/fb_BackGauge.TcPOU`

已确认状态：

- `BackGauge_AtHome := (BackGauge_T1_ActPos - BackgaugeMaxPos) < CONST_BACKGAUGE_POS_Window`
- `Table_AtHome := Table_ActPos < 5.0`
- `Feed_AtHome := Feed_ActPos > -100.0 AND Feed_ActPos < 5.0`
- `Feed_AtWork := Feed_ActPos < -300.0`

结论：

- 后挡、台面、送料已经具备“初始位/工作位”状态定义。
- `TaskBackGauge / TaskTable / TaskFeed` 已能驱动回位任务，整机复位可直接复用。

### 2.2 翻板

文件：

- `26.4/Tc_JSZW400/JSZW/Untitled1/POUs/UNIT/fb_Apron/fb_Apron.TcPOU`
- `26.4/Tc_JSZW400/JSZW/Untitled1/POUs/UNIT/fb_Apron/FB_FoldAbsPos.TcPOU`

已确认状态：

- `ApronTopSlide_AtWork`
- `ApronBtmSlide_AtWork`
- `ApronTopFold_AtHome`
- `ApronBtmFold_AtHome`

已确认互锁：

- 翻板滑动/折弯输出都要求 `Slitter_AtHome`
- 顶/底翻板折弯要求对应工作侧滑台到位
- 部分动作还要求钳口高度满足限制

结论：

- 翻板复位不能简单并发，必须按“滑台工作侧切换 -> 对应翻板折回原点”的顺序走。
- 分条不到原位时，翻板复位不应启动。

### 2.3 钳口

文件：

- `26.4/Tc_JSZW400/JSZW/Untitled1/POUs/UNIT/fb_Clamp/fb_Clamp.TcPOU`

已确认状态：

- `Clamp_AtPressPoint`
- `Clamp_LtHalfPoint`
- `Clamp_AtZeroPoint`

已确认互锁：

- `ClampUnlock` 依赖 `Slitter_AtHome`、上下翻板到原点
- `ClampLock` 还额外依赖 `Table_AtHome OR Flip_AtWork`
- 部分夹钳动作依赖送料位置

结论：

- 钳口不是单独设备动作，复位顺序必须在分条、翻板、台面、送料边界内统一考虑。
- 先解锁到安全位，再做其他轴类回位更稳妥。

### 2.4 分条

文件：

- `26.4/Tc_JSZW400/JSZW/Untitled1/POUs/UNIT/fb_Slitter/FB_SlitterAct.TcPOU`
- `26.4/Tc_JSZW400/JSZW/Untitled1/POUs/UNIT/fb_Slitter/fb_Slitter.TcPOU`
- `26.4/Tc_JSZW400/JSZW/Untitled1/DUTs/DUTs/E_UnitTask.TcDUT`

已确认现状：

- `FB_SlitterAct` 里“回原位”仍直接吃 `Hmi_bArray[44]`
- `fb_Slitter` 目前只识别 `TaskSlitter = eTask_SlitterAct`
- `E_UnitTask` 里没有独立 `SlitterHome` 任务

结论：

- 分条回原位是整机复位链路里的主要缺口。
- 如果不先任务化，整机复位只能继续绕回 HMI 位触发，不利于 PLC 内部顺序控制。

## 3. 整机复位建议初始状态

以下口径可作为后续实施和验收基线：

1. 钳口处于解锁/安全打开位，且不保持持续输出。
2. 后挡在 `BackGauge_AtHome`。
3. 台面在 `Table_AtHome`。
4. 送料在 `Feed_AtHome`。
5. 上下翻板折弯角回原点，滑台回约定安全位。
6. 分条在 `Slitter_AtHome`。
7. 自动、半自动、分条自动等当前任务步骤不再继续执行。

## 4. 建议复位顺序

基于当前代码结构，建议顺序如下：

1. 上升沿触发后先清空现有 `TaskClamp / TaskBackGauge / TaskApron / TaskSlitter / TaskTable / TaskFeed`。
2. 钳口先解锁到安全位。
3. 后挡、台面、送料回原位。
4. 分条回原位。
5. 切换滑台工作侧并将上下翻板分别折回原点。
6. 最终回到默认安全姿态并清空整机复位步骤。

## 5. 关键互锁

代码中已能明确确认的互锁：

- 分条不到原位，翻板相关动作不得执行。
- 翻板不到原位，钳口解锁/锁紧链路不应误动作。
- 台面、送料位置参与钳口动作放行。

必须现场确认的互锁：

- 各原点信号在真实机构上的可靠性。
- 出料平台或现场附加机构是否需要纳入整机复位。
- 急停、油压、润滑、变频器报警介入时，整机复位应否直接阻断。

## 6. 异常恢复建议

1. 任一工位超时未到位时，整机复位应停在当前步骤并给出明确工位名。
2. 不允许某一步失败后仍继续推进后续复位步骤。
3. 急停解除后不得自动续跑整机复位，必须重新发起。
4. 物理复位按钮与 HMI 复位如果逻辑不一致，必须在现场定一套唯一口径。

## 7. 离线可确认项与现场必须确认项

离线可确认：

- 状态位定义是否存在。
- 任务总线是否存在。
- 主要互锁是否编码在单元 FB 中。
- 分条回原位是否仍是 HMI 位直驱。

现场必须确认：

- 各工位真实初始位。
- 复位顺序的机械安全性。
- 原点/接近开关可靠性。
- 未建模机构是否影响整机复位。
