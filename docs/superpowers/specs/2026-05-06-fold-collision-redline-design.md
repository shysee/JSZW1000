# 折弯碰撞红线规则设计

日期：2026-05-06

## 背景

折弯预览已经先生成当前帧板型点列，再由相邻点绘制折弯图形线段。红线最终也是按线段编号绘制。因此规则需要明确：碰撞判定主体是点还是线段，以及碰撞命中后如何映射到红线段。

## 已确认设计

红线判定以当前帧板型点列为输入。

- 判定主体：点列中的已成型折弯点
- 输出主体：与碰撞点直接相邻的线段
- 当前折弯点：不参与碰撞点判定
- 当前折弯工作边：不加入红线集合

## 规则

1. 取得当前帧点列 `currentPreviewPolyline`。
2. 确定当前折弯点 `anchorIndex`。
3. 按当前步的右侧成型图形方向遍历点。
4. 跳过当前折弯点。
5. 对已成型折弯点判断是否进入当前碰撞区。
6. 若点命中碰撞区，只添加该点直接关联的两条线段：
   - `pointIndex - 1`
   - `pointIndex`
7. 添加线段前排除当前折弯工作边。
8. 不使用线段中点、线段采样或整段相交作为主红线判定。

## 碰撞区

碰撞点命中的区域沿用现有碰撞区定义：

- 送料完成态：固定送料区、上钳口边界、当前翻板判定区。
- 折弯完成态：上下折弯边界围成的右向角域。
- FLIP 完成态：不做工作翻板硬碰撞判定。

## 设计理由

点列是折弯图形的原始表达。用点作为判定主体，可以直接对应“哪个折弯点进入碰撞区”；再将该点相邻线段标红，能提示操作者是哪一处折弯结构导致风险。

当前折弯点和当前折弯工作边必须排除，因为它们是本步折弯动作的允许边。真正风险来自再前一个点及更早的已成型折弯点。
